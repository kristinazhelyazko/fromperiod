# Telegram Bot для учета позиций в магазине

Telegram бот и веб-приложение для учета позиций в магазине с функциями пересчета позиций и оформления привоза.

## Установка и запуск

### Шаг 1: Установка зависимостей

Откройте терминал в корневой директории проекта (`C:\pstock`) и выполните:

```bash
npm install
```

### Шаг 2: Настройка базы данных

1. Убедитесь, что PostgreSQL установлен и запущен
2. Создайте базу данных:
```sql
CREATE DATABASE pstock;
```

### Шаг 3: Настройка переменных окружения

Файл `.env` уже создан в корне проекта. Откройте его и заполните:

- **DATABASE_URL** - строка подключения к PostgreSQL (замените `postgres`, `password`, `localhost`, `5432`, `pstock` на ваши данные)
- **TELEGRAM_BOT_TOKEN** - токен бота от @BotFather
- **ERROR_CHANNEL_ID** - ID канала для ошибок (формат: `-1001234567890`)
- **ORDER_CHANNEL_ID** - ID канала для заказов (формат: `-1001234567890`)
- **SESSION_SECRET** - случайная строка для безопасности (можно сгенерировать любую)
- **WEB_APP_URL** - URL вашего веб-приложения (пока можно оставить `https://www.google.com`)

### Шаг 4: Выполнение миграций

**В корневой директории проекта** (`C:\pstock`) выполните:

```bash
npm run migrate
```

Эта команда создаст все необходимые таблицы в базе данных.

### Шаг 5: Создание первого администратора

**В корневой директории проекта** (`C:\pstock`) выполните:

```bash
npm run create-admin
```

Вам будет предложено ввести:
- Логин для администратора
- Пароль для администратора

**Важно:** Запомните эти данные, они понадобятся для входа в бота!

### Шаг 6: Запуск приложения

Для работы приложения нужно запустить **два процесса одновременно**:

#### Вариант 1: Запуск в двух терминалах

**Терминал 1 - Telegram бот:**
```bash
cd C:\pstock
npm start
```

**Терминал 2 - API сервер:**
```bash
cd C:\pstock
npm run start:server
```

#### Вариант 2: Режим разработки (с автоперезагрузкой)

**Терминал 1 - Telegram бот:**
```bash
cd C:\pstock
npm run dev
```

**Терминал 2 - API сервер:**
```bash
cd C:\pstock
npm run dev:server
```

### Проверка работы

1. **Бот:** Найдите вашего бота в Telegram и отправьте команду `/start`
2. **Сервер:** Откройте в браузере `http://localhost:3000/health` - должно вернуться `{"status":"ok"}`

## Структура команд

Все команды выполняются **в корневой директории проекта** (`C:\pstock`):

- `npm install` - установка зависимостей
- `npm run migrate` - выполнение миграций БД
- `npm run create-admin` - создание администратора
- `npm start` - запуск бота
- `npm run start:server` - запуск API сервера
- `npm run dev` - запуск бота в режиме разработки
- `npm run dev:server` - запуск сервера в режиме разработки

## Первый запуск

После установки и настройки:

1. Запустите бота
2. Найдите вашего бота в Telegram
3. Отправьте команду `/start`
4. Введите логин и пароль администратора, созданного через `npm run create-admin`
5. После авторизации вы увидите меню с функциями бота

## Функциональность бота

### Авторизация
- Команда `/start` - начало работы, запрос логина и пароля
- Проверка прав доступа (сотрудник/администратор)

### Меню пользователя
- **Перейти в приложение** - ссылка на веб-приложение (доступно всем)
- **Создать отчет** - генерация Excel отчета за месяц (только администратор)
- **Добавить пользователя** - создание нового пользователя (только администратор)

### Функции администратора
- Добавление новых пользователей с выбором прав доступа
- Генерация отчетов за выбранный месяц (будет реализовано)

## База данных

Проект использует PostgreSQL. Все таблицы создаются автоматически при выполнении миграций.

Основные таблицы:
- `rights` - права доступа
- `users` - пользователи
- `address` - адреса магазинов
- `category` - категории товаров
- `item` - позиции товаров
- `stockreport` - отчеты пересчета
- `stock` - записи пересчета
- `replenish` - привозы
- `replenishstock` - позиции привоза

## Разработка

Для разработки используется `nodemon` для автоматической перезагрузки при изменении файлов.

Каталоги `backend/` и `frontend_client/` — приложения на Next.js: после клонирования репозитория выполните в каждом из них `npm install` и `npm run build` (артефакты `.next/` в Git не хранятся — они создаются при сборке и указаны в `.gitignore`).

## Логирование

Все ошибки логируются в файлы `error.log` и `combined.log`, а также отправляются в канал ошибок (если настроен).
