# Настройка домена fromperiod.ru для скрытия «Тестовый магазин»

## Проблема

При открытии https://fromperiod.ru запросы приходят **не на этот сервер** (ответ 404 на /api/addresses-debug).  
Поэтому правки в коде и БД на этом сервере не влияют на то, что видит пользователь в Telegram.

## Что уже сделано на этом сервере

- В БД у «Тестовый магазин» установлено `is_visible_in_store = false`.
- API `GET /api/addresses` отдаёт только видимые адреса (Белгород, Северный, Строитель).
- В Next.js (client_frontend) добавлены маршруты `/api/addresses` и `/api/addresses-debug`, которые проксируют запросы на наш web.

## Вариант A: Домен ведёт на этот сервер

1. Убедиться, что DNS для fromperiod.ru указывает на IP этого сервера.
2. Настроить обратный прокси (nginx/caddy на хосте):
   - Запросы на `https://fromperiod.ru` и `https://fromperiod.ru/api/*` проксировать на **localhost:3002** (контейнер client_frontend).
3. Перезапустить контейнеры при необходимости и проверить: https://fromperiod.ru/api/addresses-debug должен вернуть JSON с полем `"via": "next-api-route"`.

После этого приложение в Telegram будет получать список адресов с этого сервера, и «Тестовый магазин» не будет отображаться.

## Вариант B: Домен остаётся на текущем хостинге (например, Vercel)

1. Сделать доступным наш API по публичному URL, например:
   - поддомен: `https://api.fromperiod.ru` → прокси на этот сервер, порт 3000 (web), или
   - тот же сервер по IP и порту, если есть доступ с интернета.
2. В проекте, который деплоится на fromperiod.ru (тот же frontend_client или его копия), задать базовый URL API так, чтобы клиент ходил на наш сервер:
   - либо в сборке: `NEXT_PUBLIC_API_BASE_URL=https://api.fromperiod.ru` (и на клиенте использовать его в `window.__API_BASE_URL__` или в `apiRequest`),
   - либо в коде клиента: `API_BASE_URL = 'https://api.fromperiod.ru'` вместо `window.location.origin + '/api'`.
3. Задеплоить обновлённый фронтенд на текущий хостинг fromperiod.ru.

Тогда запросы к `/api/addresses` пойдут на наш web, и «Тестовый магазин» снова не будет отображаться.

## Проверка

- Открыть https://fromperiod.ru/api/addresses-debug: если виден JSON с `"via": "next-api-route"` или ответ от нашего web — запросы доходят до этого сервера.
- Открыть магазин в Telegram: в выборе пунктов должны быть только Белгород, Северный, Строитель.
