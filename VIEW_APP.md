# Запуск приложения и просмотр Web App

## Подготовка окружения

1. Откройте терминал в директории проекта:
   ```bash
   cd C:\pstock
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Настройте файл .env (в корне проекта). Минимально:
   ```env
   TELEGRAM_BOT_TOKEN=ваш_токен_бота
   DATABASE_URL=postgres://user:password@localhost:5432/pstock
   WEB_APP_URL=http://localhost:3000
   ERROR_CHANNEL_ID=123456789        # необязательно, для ошибок
   REPORT_CHANNEL_ID=123456789       # канал для отчетов и привозов
   ORDER_CHANNEL_ID=123456789        # канал для заказов (если отдельно)
   PORT=3000                         # порт веб-сервера
   ```

4. Выполните миграции:
   ```bash
   npm run migrate
   ```

5. Создайте первого администратора:
   ```bash
   npm run create-admin
   ```
   Введите логин и пароль администратора.

## Запуск

Запуск необходим в двух терминалах.

1. Терминал 1 — Telegram бот:
   ```bash
   cd C:\pstock
   npm start
   ```

2. Терминал 2 — API сервер:
   ```bash
   cd C:\pstock
   npm run start:server
   ```
   
   Сервер запустится на `http://localhost:3000`

Альтернатива для разработки (с автоперезагрузкой):
- Бот: `npm run dev`
- Сервер: `npm run dev:server`

## Просмотр в браузере

 - Откройте браузер (Chrome, Firefox, Edge)
 - Перейдите по адресу: `http://localhost:3000`
 - Вы увидите главное меню приложения

## Открытие через Telegram Web App

1. Убедитесь, что запущены оба процесса (бот и сервер).

2. Настройте WEB_APP_URL в .env:
   Откройте файл `.env` и укажите URL вашего сервера:
   ```env
   WEB_APP_URL=http://localhost:3000
   ```
   
   Или если у вас есть публичный URL:
   ```env
   WEB_APP_URL=https://your-domain.com
   ```

3. Откройте через Telegram:
   - Найдите вашего бота в Telegram
   - Отправьте команду `/start`
   - Войдите с логином и паролем администратора
   - Нажмите кнопку **"📱 Перейти в приложение"**
   - Приложение откроется внутри Telegram

## Прямая ссылка на Web App

Если сервер запущен, вы можете открыть приложение напрямую:
- `http://localhost:3000` - главная страница приложения

## Проверка работы

### Проверка сервера:
Откройте в браузере: `http://localhost:3000/health`
Должно вернуться: `{"status":"ok","timestamp":"..."}`

### Проверка API:
- `http://localhost:3000/api/addresses` - список адресов
- `http://localhost:3000/api/categories/1` - категории для адреса с ID=1

## Важные замечания

1. Для работы через Telegram:
   - Бот и сервер должны быть запущены одновременно
   - URL в `.env` должен быть доступен из интернета (или используйте ngrok для локальной разработки)

2. Для локальной разработки через Telegram:
   Используйте ngrok для создания публичного URL:
   ```bash
   ngrok http 3000
   ```
   Затем укажите полученный URL в `.env`:
   ```env
   WEB_APP_URL=https://your-ngrok-url.ngrok.io
   ```
   Telegram требует HTTPS для Web App. Если указать `http://` (не localhost), кнопка будет заменена на обычную URL‑кнопку.

3. CORS настройки:
   Если возникают проблемы с CORS, убедитесь, что в `server.js` правильно настроен CORS для Telegram Web App.

## Устранение проблем

**Приложение не открывается:**
- Проверьте, что сервер запущен (`npm run start:server`)
- Проверьте, что порт 3000 не занят другим приложением
- Проверьте консоль браузера на наличие ошибок (F12)

**Кнопка "Перейти в приложение" не работает:**
- Убедитесь, что `WEB_APP_URL` правильно указан в `.env`
- Перезапустите бота после изменения `.env`
- Проверьте, что URL доступен из интернета

**Ошибки API:**
- Убедитесь, что база данных запущена и миграции выполнены
- Проверьте `DATABASE_URL` в `.env`
- Проверьте логи сервера на наличие ошибок

**Права администратора не применяются:**
- Убедитесь, что пользователь вошел успешно перед нажатием кнопки в меню
- Проверьте, что у пользователя роль в таблице `rights` — «администратор»
- Перезапустите бота, если меняли права вручную в БД
