# Исправление проблемы с WEB_APP_URL

## Проблема
Telegram требует **HTTPS** для Web App URL. Если вы указали `WEB_APP_URL=http://localhost:3000`, бот выдаст ошибку:
```
Only HTTPS links are allowed
```

## Решение

### Вариант 1: Использовать ngrok (рекомендуется для разработки)

1. **Установите ngrok:**
   - Скачайте с https://ngrok.com/download
   - Или через npm: `npm install -g ngrok`

2. **Запустите ngrok в отдельном терминале:**
   ```bash
   ngrok http 3000
   ```

3. **Скопируйте HTTPS URL** (например: `https://abc123.ngrok.io`)

4. **Обновите `.env`:**
   ```env
   WEB_APP_URL=https://abc123.ngrok.io
   ```

5. **Перезапустите бота:**
   ```bash
   npm start
   ```

### Вариант 2: Использовать обычную URL кнопку (для локальной разработки)

Если вы используете `http://localhost:3000`, бот автоматически заменит Web App кнопку на обычную URL кнопку. При нажатии откроется браузер с вашим приложением.

**Важно:** Это работает только для локальной разработки. В продакшене нужен HTTPS.

### Шаг 1: Проверьте файл .env
Убедитесь, что в файле `.env` указано:
```env
WEB_APP_URL=http://localhost:3000
```
**Важно:** Не должно быть пробелов вокруг знака `=`

### Шаг 2: Перезапустите бота
После изменения `.env` файла **обязательно перезапустите бота**:

1. Остановите бота (Ctrl+C в терминале)
2. Запустите снова:
   ```bash
   npm start
   ```

### Шаг 3: Проверьте логи
После перезапуска в логах должно появиться сообщение:
```
Using WEB_APP_URL: http://localhost:3000
```

Если вы видите `Using WEB_APP_URL: https://www.google.com`, значит переменная не читается.

### Шаг 4: Проверьте, что сервер запущен
Убедитесь, что API сервер запущен на порту 3000:
```bash
npm run start:server
```

### Шаг 5: Проверьте в Telegram
1. Откройте бота в Telegram
2. Отправьте `/start`
3. Войдите в систему
4. Нажмите кнопку "📱 Перейти в приложение"
5. Должно открыться ваше приложение на `http://localhost:3000`

## Важные замечания

### Для локальной разработки через Telegram:
Telegram Web App требует **HTTPS** или специальную настройку. Для локальной разработки:

1. **Используйте ngrok** для создания публичного HTTPS URL:
   ```bash
   ngrok http 3000
   ```
   
2. Скопируйте полученный URL (например: `https://abc123.ngrok.io`)

3. Обновите `.env`:
   ```env
   WEB_APP_URL=https://abc123.ngrok.io
   ```

4. Перезапустите бота

### Альтернатива: Прямой доступ
Если вы просто хотите протестировать приложение локально, откройте в браузере:
```
http://localhost:3000
```

## Отладка

Если проблема сохраняется:

1. Проверьте, что файл `.env` находится в корне проекта (`C:\pstock\.env`)
2. Убедитесь, что нет лишних пробелов или кавычек в значении
3. Проверьте логи бота на наличие ошибок
4. Убедитесь, что `dotenv` установлен: `npm list dotenv`
