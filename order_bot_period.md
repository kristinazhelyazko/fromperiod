# PROMPT ДЛЯ TRAE — Реализация order-бота с напоминаниями

Ты — опытный backend / Telegram Bot разработчик.  
Твоя задача — **реализовать функционал учёта заказов и напоминаний для цветочного магазина**, строго **в рамках уже существующего Telegram-бота**.

## КОНТЕКСТ
- Бот уже существует.
- В боте уже есть:
  - логирование;
  - определение роли пользователя (admin / employee);
  - подключение к PostgreSQL (Supabase).
- Текущую архитектуру, кодовую базу и стиль реализации **ломать нельзя** — только расширять.

## ЦЕЛЬ
Устранить проблему забытых отложенных заказов за счёт:
- строгого учёта заказов;
- автоматических напоминаний;
- удобного интерфейса через Telegram-кнопки.

## ОБЩИЕ ТРЕБОВАНИЯ
- Используй существующие middleware, репозитории, ORM / SQL-подход.
- Минимизируй ручной текстовый ввод.
- Используй inline-кнопки везде, где это возможно.
- Код должен быть расширяемым и читаемым.
- Все новые таблицы — в PostgreSQL (Supabase).

## ФУНКЦИОНАЛ 1. ГЛАВНОЕ МЕНЮ
Добавь кнопки:
- Создать заказ
- Управление заказами

Доступны для ролей: admin, employee.

## ФУНКЦИОНАЛ 2. СОЗДАНИЕ ЗАКАЗА
Пошаговый сценарий:

1. Тип получения: Самовывоз / Доставка  
2. Точка магазина: Северный / Строитель / Белгород  
3. Дата выполнения: inline-календарь (без прошедших дат)  
4. Время выполнения: кнопки (шаг 30–60 минут)  
5. Тип заказа:
   - Свадебный букет
   - Композиция
   - Еда
   - Цветы + еда
   - Другое

## ФУНКЦИОНАЛ 3. ДЕТАЛИ ЗАКАЗА
- Свадебный букет: состав, до 2 фото, бутоньерка (да/нет)
- Композиция: коробка/букет, состав, цвет (опц.), до 2 фото (опц.), открытка
- Еда: позиции, открытка
- Другое: свободное описание

## ФУНКЦИОНАЛ 4. КОНТАКТЫ
- Имя и телефон клиента
- Имя и телефон получателя
- Адрес получателя (если доставка)

## ФУНКЦИОНАЛ 5. ПОДТВЕРЖДЕНИЕ
- Одно итоговое сообщение
- ID заказа
- Кнопки: Все корректно / Внести изменения
- Возможность редактирования полей
- После подтверждения: status = active, планирование напоминаний

## ФУНКЦИОНАЛ 6. УПРАВЛЕНИЕ ЗАКАЗАМИ
- Список активных заказов
- Карточка заказа
- Кнопка «Выполнено» → status = completed

## ФУНКЦИОНАЛ 7. НАПОМИНАНИЯ
- Отправка в отдельный Telegram-канал
- Триггеры: за 7 дней, за 1 день, в день выполнения
- Формат сообщения:

Напоминание!
Есть активный заказ №{order_id} на {execution_date}.

Количество дней: {days_left}

Для просмотра деталей заказа выберите функцию
"Управление заказами" и заказ №{order_id}.

## ФУНКЦИОНАЛ 8. БАЗА ДАННЫХ
Таблицы:
- users (уже существует)
- orders
- order_details
- order_photos
- contacts
- reminders

Обеспечь связи и защиту от дубликатов напоминаний.

## ФУНКЦИОНАЛ 9. ПЛАНИРОВЩИК
- background job
- ежедневная проверка reminders
- отправка непросланных
- sent = true после отправки

## КРИТЕРИИ ГОТОВНОСТИ
- Заказы не забываются
- Напоминания приходят вовремя
- Выполненные заказы не активны
- Интеграция без конфликтов с текущим ботом
