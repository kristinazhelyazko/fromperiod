Составь план разработки для следующего проекта в md файле. Нужен бот в телеграмме и веб-приложение в телеграмме для учета позиций в магазине. 
Задача бота: проверка логина и пароля сотрудника, в зависимости от прав (админ или сотрудник) выдавать кнопки: переход в приложение (сотрудник, админ), сделать отчет (только админ), добавить пользователя. 
Как формируется отчет объясню позже. 
Функции веб-приложения. 
При переходе в приложение нужно давать выбор функции "оформление привоза", "пересчет позиций".
Пересчет позиций:
При выборе функции пересчет позиций выводится кнопки с адресами магазинов (Address.Name). Адреса будут храниться в таблице Address. Записи будут следующие(Address.Name): Белгород, Северный, Строитель.
После выбора адреса требуется выбрать одну из категорий товара из таблицы Category.Name. Записи будут: бар, цветы, кухня, цех. Должны выводится только те записи, которые есть по этому адресу. Для этого в таблице Category должна быть ссылка на таблицу Address и на странице выбора раздела должны выводится только те записи, которые есть на этом Address.
После выбора раздела выводится таблица с позициями из этого раздела. Я предлагаю сделать таблицу Item и сделать ссылку на раздел и Address. В таблице Item делаем столбец expected для того, чтобы регулировать сколько должно быть позиций для каждой позиции и каждого Address. 
В этой таблице в интерфейсе будет 3 столбца: Наименование, Количество, Требуемое. Столбцы наименование и должно быть должны быть заблокирвоаны для редактирования.
В таблицы ниже должна быть кнопка Добавить для создания новых записей. Тогда при добавлении новой записи эта позиция должна быть добавлена новая запись в item с ссылкой на категорию, но в address_id=0, чтобы это наименование не добавлялось в следующий раз. 
Пользователь должен заполнить поле Количество, поле необязательное для заполнения, так как каких то позиций может не быть вообще, тогда это поле заполняется 0 в таблице Stock. В таблице должны быть ссылка на item, address, stockreport, а также дата, количество. 
Ниже таблицы должна быть кнопка "Сохранить". После нажатия на кнопку сохранить кнопку нужно заблокировать, а также заблокировать таблицу со всеми позициями, должно выводится уведомление об успешном сохранении или об ошибке, если была ошибка сохранения записей. После первого сохранения создается новая запись в таблице stockreport. Разблокировать кнопки "Сформировать заказ", "Изменить". При нажатии на кнопку изменить нужно давать таблицу редактировать. После сохранения обновляем записи.
Таблица stockreport нам нужна, чтобы в один день мог создаваться только один пересчет позиций, если пользователь отредактирует таблицу, то мы сначала ищем по StockReport_id и обновляем записи только в тех записях таблицы Stock, у которых ссылка на этот StockReport. Давай еще внизу таблицы, но выше кнопок выводить "Номер отчета: [StockReportId]".
При нажатии на кнопку Сформировать заказ ниже генерируется заказ. Он должен формироваться следующим образом: для каждой позиции нужно найти его в таблице Item с необходимым Address_id и высчитать сколько не хватает позиций с помощью поля Expected. 
Позиции должны выводится по порядку, сверху должен быть написан раздел (цветы, бар, цех, кухня). 
Можно сделать общий блок, внутри которого будут отображаться все необходимые позиции. Те позиции, которые >= expected, выводить не нужно. 
Нужно сделать еще кнопку, по которой будет копироваться весь отчет в буфер обмена. 
После генерации этот отчет автоматически отправляется в отдельный канал, куда будет добавлен  наш бот. 
Нужно реализовать кнопку Назад на всех экранах. Если пересчет не был сохранен, то спрашиваем сначала у пользователя, точно ли он хочет удалить пересчет, если ответ положительный, то выходим на предыдущий экран. 
Оформление привоза: 
При выборе этой функции оформление привоза выводится кнопки с адресами магазинов (Address.Name). Пользователь выбирает кнопку - адрес. 
Дальше выбирает раздел из Category.Name. После выводится поле-справочник со всеми Item, которые есть для этого address_id. Пользователь может ввести наименование в поле, к примеру он выбрал категорию Цветы, теперь в поле вводит "Роза". В справочнике апи посылает запрос только при вводе 3 символов, чтобы не грузить базу данных. И в поле-справочнике будут выводится все наименование, где есть %роза%. 
При выборе конкретной записи из справочника она добавляется в таблицу ниже. Таблица нужна из двух столбцов: наименование (выбранная запись из item), количество. 
Поле наименование заблокировано в таблице и недоступно для редактирования. Добавление записей из справочника в таблицу не ограничиваем. 
Ниже таблицы кнопка сохранить. При нажатии на кнопку сохранить создается запись в таблице replenish, где есть ссылка на address_id. В отдельной таблице replenishstock, где есть ссылки на item_id и replenish_id и поле qty.
После нажатия на кнпоку сохранить  ниже появляется запись "Номер привоза: [replenish_id]". После сохранения появляется уведомление об успешной отправке или ошибке. После сохранения ниже создаем список для копирования по аналогии с отчетом по пересчету. Используем одни и те же компоненты и стили для единобразия. 
При изменении каких-то записей и повторном сохранении привоза обновляются записи в replenishstock с этим replenish_id. Создание еще одного привоза в replenish и повторное сохранение тех же записей - блокирующая ошибка. 
Функции чат_бота. 
Проверка логина, пароля: первый шаг для использования бота - логирование. Запрашивается логин, осуществляется поиск в базе данных в таблице Users.Login. Если запись с таким Users.Login есть, то запрашиваем пароль, проверяем его с полем Password в базе данных для этого логина.
В таблице Users нужна ссылка на таблицу rights. В этой таблице нам нужны записи "сотрудник", "администратор". В Users будет лежать ссылка на rights_id, так мы будем регулировать права доступа. 
У пользователя сотрудник будет возможность только перехода в приложение. 
У пользователя с правами администратора будет возможность создания новых пользователей. При нажатии этой кнопки запрашивается логин (пользователю нужно ввести текст), пароль (пользователю нужно ввести текст), выбрать права доступа (нужно выбрать одну из двух кнопок: сотрудник, администратор). 
После этого будет создаваться новый пользователь в таблице Users. 
У пользователя с правами администратор еще будет возможность выбора кнопки "Создать отчет". При ее нажатии пользователю требуется выбрать временной диапазон в виде кнопок по месяцам - должны отображаться последние 6 месяцев, включая текущий. 
После выбора месяца создается excel файл со следующим наполнением: 1 лист - привозы за период (столбцы: наименование, дата, количество, итоговая сумма; если эта позиция была привезена несколько раз за период, то во втором столбце ячейка наименование объединяет столько дат и количеств, сколько было осуществлено, итоговая сумма - количество привезенных единиц товара за весь период), 2 лист - пересчеты позиций за этот период (столбцы аналогичные листу привозов).
Если файл начал генерироваться, то присылаем пользователю сообщение о просьбе ожидания, задача выполянется. Если есть ккая то ошибка - то ее вывести пользователю в чат. 
Все ошибки в работе приложения, бота должны  отправляться мне в отдельный канал.
